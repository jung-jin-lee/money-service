핵심 문제해결 전략

```
1. 분배 로직

예를 들어,

50000원을 3명한테 뿌리기를 하면,
딱 맞게 금액이 떨어지지 않습니다.

그래서 일단 50000/3 으로 나누고 소수점은 버리는 전략을 취했습니다.
그리고 분배가 되지 않은 금액은 한 명에게 분배하는 방식으로
분배 로직을 간소화하였습니다.
```

```
2. 엔티티 구조
---------------------      -----------------------------
| 뿌리기(Sprinkling) | 1..n | 뿌려진 항목(SprinklingItem) | 
---------------------      -----------------------------
```

```
3. 뿌린 건 유효 값 체크
뿌리기 생성 필드(createdAt) 을 활용해서 유효 값을 체크하였고
createdAt 은 엔티티 생성 시 셋팅해주는 방식으로 해서
실제 딜레이를 주지 않고도 테스트를 하도록 하였습니다.
```
